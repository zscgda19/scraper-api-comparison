## ScraperAPI 是什么，做促销监控为什么会用到它

做电商选品、竞品价格追踪、促销活动监控，绕不开一个问题：目标网站的反爬机制。

ScraperAPI 本质上是一个代理 + 浏览器渲染 + IP 轮换的集成服务。你发一个 HTTP 请求，它帮你处理 IP 封锁、User-Agent 检测、JavaScript 渲染、验证码绕过这些脏活。你拿到的是干净的 HTML，直接解析就行。

对于促销监控这个场景，它的价值点很具体：

- **价格页面大多是 JS 动态渲染的**，普通 requests 库拿不到真实价格，ScraperAPI 的 `render=true` 模式直接解决这个问题
- **高频抓取会触发 IP 封禁**，它的住宅 IP 池和自动轮换机制让你不用自己维护代理池
- **地理定价越来越普遍**，`country_code` 参数可以指定抓取节点所在国家，拿到对应地区的促销价格

我自己的用法是：用 Python脚本每小时调用一次，把目标 SKU 的价格写进数据库，价格变动超过 5% 就触发 Slack 通知。整套流程跑了快八个月，ScraperAPI 这一层基本没出过问题。

## 核心功能拆解：促销监控场景最常用的几个参数

### 基础抓取 vs 渲染模式

不带 `render=true` 的请求走的是纯 HTTP 模式，速度快、消耗积分少，适合静态页面或者服务端渲染的老站点。

带上 `render=true` 之后，ScraperAPI 会启动无头浏览器，等 JS 执行完再返回 DOM。价格监控里，大概 60% 的目标站点需要这个模式。代价是每次请求消耗 5 个积分而不是 1 个，规划用量的时候要算进去。

### Premium Proxies（高级代理）

普通代理池搞不定的站点，加 `premium=true` 参数会切换到住宅 IP 池。这个参数对付 Amazon、Walmart 这类反爬力度大的平台效果明显。消耗积分是普通模式的 10–25 倍，但比自己买住宅代理便宜得多。

### 地理定位抓取

```
https://api.scraperapi.com/?api_key=YOUR_KEY&url=TARGET_URL&country_code=de
```

`country_code` 支持 50+ 个国家。做跨境选品的时候，同一个商品在德国站和美国站的促销力度可能差很多，这个参数直接解决了需要多套代理账号的问题。

### Structured Data Endpoints

这是我最近才开始用的功能。ScraperAPI 针对 Amazon 产品页、搜索结果页提供了结构化数据接口，直接返回 JSON，不用自己写解析器。对于只监控 Amazon 促销的场景，这个功能能省掉大量维护 XPath 的时间。
